System Design

Dynamic Scoring Engine

External judges needed to evaluate hundreds of applicants across multiple programmes, each with different criteria and scoring rubrics. The previous process was manual spreadsheets — slow, error-prone, and didn't scale.

Approach

Designed an algorithm-driven form engine with configurable point-scoring logic. Forms are dynamically generated from a schema that defines fields, weights, and scoring rules. Judges are assigned forms with granular permissions, and scores aggregate in real-time across evaluators.

Admin Config → Form Schema Engine → Dynamic Forms → Score Aggregator → Results Dashboard
Node.js + Express / MySQL / React
~60% reduction in manual evaluation effort
AWS Migration

Legacy Platform to Modern AWS Stack

Inherited a legacy codebase from a third-party vendor with 15+ critical production bugs, no documentation, and recurring outages. The platform needed to be stabilised immediately while planning a migration to a modern, maintainable architecture.

Approach

First stabilised production by triaging and fixing all critical bugs. Then planned a phased migration to AWS — EC2 for compute, RDS for managed MySQL, S3 for assets, CloudFront for CDN, Route 53 for DNS, and SES for transactional email with proper DMARC/SPF/DKIM authentication. Nginx as reverse proxy with SSL termination. Redis for session and query caching.

CloudFront CDN → EC2 + Nginx → Node.js API → RDS MySQL
Route 53 SES (DMARC) Redis S3 CloudWatch
Full AWS stack / Zero recurring outages post-migration
Eliminated all recurring production outages
Product Feature

WYSIWYG Content Management System

Non-technical staff needed to publish news articles, blog posts, and programme updates. Every content change required a developer, creating a bottleneck and 3-5 day turnaround for simple updates.

Approach

Infrastructure

Email Security & Deliverability

Transactional emails (password resets, notifications, programme updates) were landing in spam or not being delivered at all. No email authentication was configured, damaging sender reputation.

Approach

Configured AWS SES as the email provider with full DMARC, SPF, and DKIM authentication. Set up a custom MAIL FROM domain for SPF alignment. Added CloudWatch event tracking for bounces and complaints. Configured Nginx for proper reverse DNS. Moved DMARC policy from "none" to "quarantine" after monitoring reports.

Near-100% inbox delivery rate after configuration
